
Классификация:
• Навигационные              (навигация между записями по ссылкам)
• Реляционные (RDBMS)        (на базе отношений/связей между таблицами)
• SQL
• Объектно-ориентированные
• ORM
• NoSQL: Key-Value, Документные, Графовые, БД в памяти, Поисковые
• Hybrid
• Persistent    - постоянное надежное хранение
• In-memory     - в оперативной памяти
• Distributed   - распределенные
• Embedded      - встраиваемые
• Graph         - Графовые
• Column        - колоночные СУБД

Определения: 
• Запись - ряд
• Колонка - поле - столбец
• Домен - свой тип данных
• Внешний ключ (foreign)    - связывает таблицы
• Первичный ключ (primary)  - уникальная идентификатор записи
• Составной ключ  - из нескольких полей
• Индекс          - объект базы данных, создаваемый с целью повышения производительности поиска данных
• Триггер         - хранимые процедуры и события, вызывающие эти процедуры
• Ограничение целостности   - наложение ограничений на запись, поле, связь
• Транзакция      - несколько операций с общим началом и общим концом
• Консистентное состояние данных    - непротиворечивые данные (до и после транзакции)

SQl делится на
• DDL   - CREATE, ALTER, DROP     (создание, изменение, удаление таблицы)
• DML   - INSERT, UPDATE, DELETE  (создание, изменение, удаление записи)
• DQL   - SELECT
• DCL   - GRANT      (работа с пользователями, правами)
• TCL   - COMMIT     (TCL-команды: работа с транзакциями)

Масштибирование:
• Репликация    - синхронизация копий БД
• Миграция      - переход к новой структуре БД
• Партиционирование (секционирование)   - разделение БД с физически разным хранением
• Шардинг       - разделение БД между серверами
• Мультимастер  - каждый сервер изменяет (каждый узел из реплицированных является мастер-узлом)

ACID принципы:
• Atomicity (атомарность)       - целостность транзакций (последовательности изменений)
• Consistency (консистентность) - согласованность и непротиворечивость
• Isolation (изолированность)   - независимое исполнение транзакций
• Durability (стойкость)        - надежность при сбоях (происходит благодаря журналу)

Типы систем:
• OLAP (Online analytical processing)   - агрегированные данные для задач аналитики
  Данные использ-ся только для чтения. Ненужны транзакции
• OLTP (Online Transaction Processing)  - обработка транзакций в реальном времени (обычные SQL БД)

Большие данные 
• Big data    - не могут хранится в оперативной памати. Требуют параллеьной обработки.
  Три характеристики (VVV): объем, прирост (интенсивность вставки), многообразие
• Data warehouse (хранилища)  - данные только для чтения, нетранзакционные, для анализа
• Data lake (озеро данных)    - хранилище большого объема неструктурированных данных
