Cтруктура файловой системы Linux
/ - корень. Только root имеет право читать и изменять файлы в этом каталоге

/bin - (binaries) бинарные файлы пользователя.
    Исполняемые файлы (утилиты, которые могут использоваться 
    пока еще не подключен каталог /usr/)

/boot - Файлы загрузчика

/dev - (devices) Виртуальная файловая система для устройств

/etc - (etcetera) конфигурационные файлы всех программ, 
    скрипты запуска и завершения системных демонов, 
    монтирования файловых систем и автозагрузки программ

/home - Домашняя папка (каталоги всех пользователей)

/lib  - (library) Системные библиотеки. Аналог system32 в WIndows

/media - Съемные носители

/mnt - (mount) Воеменное монтирование внешних или дополнительных файловых систем

/opt - (optional) Дополнительные программы, производители которого не следуют FHS Linux

/proc - (proccess) информация о процессах и системе.
    Подсистема, динамически создаваемая ядром.

/root - домашний каталог root

/run - PID процессов, похожий на /var/run, но в отличие от него, он размещен в TMPFS, 
    а поэтому после перезагрузки все файлы теряются

/sbin - (system binaries) системные исполняемые файлы. Утилиты для обслуживания системы.

/srv - (server) файлы серверов и сервисов

/sys (system) - Информация о системе, о распознаном железе

/tmp - (temp) Временные файлы

/usr        - (Unix System Resourses) вторичная иерархия программного обеспечения.
    Софт, кот. можно потом расшарить по сети.
/usr/bin/   - дополнит. программы для всех пользователей 
/usr/include - заголовочные (header) файлы
/usr/lib/   - библиотеки для программ из /usr/bin или /usr/sbin
/usr/local  - третичная иерархия программного обесп-я. Здесь то, что мы сами собираем из исходников
/usr/sbin/  - дополн. программы для админа
/usr/share  - архитектурно-независимые данные. Здесь же поддержка разных языков
/usr/src    - исходные коды программного обесп-я

/var - (variable) Часто изменяемые файлы. Кеши, бд, журналы и др.
/var/log - Файлы логов
/var/lib - базы данных
/var/mail - почта
/var/spool - очереди
/var/lock - файлы блокировок
/var/run - PID процессов. В отличие от каталога /run данные сохраняются после перезагрузки

Программа - исполняемый файл (ELF, скрипт)

Процесс - исполняющаяся программа

IPC - межпроцес. взаимодействие (shmem, msgq, семафоры (регулир. доступ к общей памяти),
     сокеты, каналы, сигналы)

Потоки выполнения (thread (нить), NTPL, LWP). Ядром воспринимаются как 
    легковесные процессы
