
Интернет — всемирная система компьютерных сетей

Интернет —  это инфраструктура, а Веб — это сервис

Компьютеры объединяются в сеть с помощью маршрутизаторов (роутеров)

Любое устров-во в сети, имеющее IP, - это хост

IP-адрес (от англ. Internet Protocol) — уникальный числовой идентификатор устройства в сети

Для подключения компьютерных сетей в телефонные служит модем.
Модем перекодирует информацию, поступающую из нашей сети в формат, 
который можно передавать через телефонную сеть, и наоборот

Обслуживанию специальных маршрутизаторов (объединяющих сети сетей) служит провайдер.

Типы сетей:
PAN - персональная сеть (в рамках одного человека)
LAN - локальная вычислительная сеть (офис)
CAN - кампусная сеть (несколько зданий)
MAN - сеть большого города
WAN - глобальная сеть

Схемы передачи данных:
Unicast   - один одному (однонаправленное вещание)
Multicast - один многим (многоадресное вещание)
Broadcast - один всем (широковещание)

Режимы связи:
Симплекс (Simplex)        - передача в одном направлении (теле / радио)
Полудуплекс (Semi Duplex) - в один момент времени либо передает, либо принимает (рация)
Дуплекс (Duplex)          - передача и прием в обе стороны (телефон)

Брэндмауэр = Фаервол = Межсетевой экран
Порт входит в понятие "интерфейс"
VPN (Virtual Private Network) - защищенная передача данных по шифрованному каналу
Протокол - набор правил передачи данных

TCP/IP - 4-хуровневая модель передачи данных:
1. Прикладной   (Application):  HTTP, FTP, SSH, DNS, SMTP, ...
2. Транспортный (Transport):    TCP, UDP
3. Сетевой      (Network):      IP
4. Канальный    (Link):         Ethernet, IEEE 802.11 WLAN

1. Прикладной уровень:
Некоторые протоколы по умолчанию привязаны к порту:
• HTTP    80 или 8080,
• HTTPS   443
• FTP     20 (для передачи данных) и 21 (для управляющих команд),
• SSH     22

HTTP (HyperText Transfer Protocol) — «протокол передачи гипертекста»
Работает по технологии «клиент-сервер», то есть "потребитель-поставщик" 
Клиент  - инициирует соединение и посылает запрос
Сервер  - ожидает соединение и возращает сообщение с результатом

DNS: Система Доменных Имён напоминает записную книжку для веб-сайтов. 
Когда вы вводите веб-адрес в своём браузере, браузер обращается к DNS, 
чтобы найти реальный адрес веб-сайта, прежде чем он сможет его получить.

2. Транспортный уровень
TCP - протокол с подтверждение доставки (надежный)
UDP - протокол без установления соединения и подтвеждения доставки 
  (ненадежный, но быстрый. обычно используется в видео и компьютерных играх)
